Conformal mapping CAD surfaces#

A conformal lattice evaluates the same normalized TPMS or graph unit cell used in the rectangular examples through a cell map derived from a surface. The map bends and stretches logical U, V, and W; the unit cell still defines topology, and wall thickness or beam radius remains a world-unit control.

This guide covers maps built from exact CAD faces, whose native surface coordinates provide the map directly. When the surface exists only as a triangle mesh — an STL region, a scan, or a procedurally generated patch — the companion guide, Conformal mapping triangle meshes, builds the same maps from a TriangleMeshSurface. Both satisfy one ParametricSurface contract, so mm.cell_map_from_surface(...) and mm.cell_map_between_surfaces(...) accept either type in any combination; the CAD-named helpers used below are thin wrappers over the same factories.

Install the optional CAD authoring stack#

CadQuery support is optional because its CAD kernel and visualization dependencies are substantial:

pip install "pyvcad-metamaterials[cad]"

pyvcad loads CadQuery only when pv.CADModel.from_cadquery(...) is called.

How a face becomes a cell map#

For a single face, native surface coordinates provide logical U and V. Logical W follows the oriented face normal through the requested signed height. A positive height follows the face normal; a negative height maps inward.

The same logical cells can occupy a block, follow one CAD face by signed height, or span between two faces.

Logical coordinates cover [0, nu] × [0, nv] × [0, nw]. One integer interval is one unit cell. Periodic CAD directions preserve periodic map connectivity, and trimmed faces keep only cells that intersect the selected face when trim_to_face=True.

Every workflow below uses one fixed orthographic camera for its CAD body, selected-face view, and mapped result. The triptychs can therefore be read as direct visual transformations rather than independently framed previews.

Author and select a CadQuery face#

CadQuery remains the high-level language for workplanes, sketches, solids, and face selectors. Convert exactly one selected face with pv.CADModel.from_cadquery(...).faces[0], then construct the map.

import cadquery as cq
import pyvcad as pv
import pyvcad_metamaterials as mm
import pyvcad_rendering as viz

cad = cq.Workplane("XY").circle(20.0).extrude(30.0)
surface = pv.CADModel.from_cadquery(cad.faces("%CYLINDER")).faces[0]

cell_map = mm.cell_map_from_cad_face(
    surface,
    cells=(24, 10, 1),
    height=4.0,
    linear=False,
)
root = mm.gyroid(cell_map, wall_thickness=0.65)

viz.Render(root)

linear=False retains curved midpoint samples instead of reducing every cell to a single trilinear element. The complete script is 12_cadquery_conformal_gyroid.py.

Map between two faces#

Use paired faces when both sides of a curvilinear volume matter. The first and second face define the W boundaries rather than using a constant normal height.

import cadquery as cq
import pyvcad as pv
import pyvcad_metamaterials as mm
import pyvcad_rendering as viz

inner_cad = cq.Workplane("XY").circle(15.0).extrude(30.0)
outer_cad = cq.Workplane("XY").circle(22.0).extrude(30.0)

inner = pv.CADModel.from_cadquery(inner_cad.faces("%CYLINDER")).faces[0]
outer = pv.CADModel.from_cadquery(outer_cad.faces("%CYLINDER")).faces[0]

cell_map = mm.cell_map_between_cad_faces(
    inner,
    outer,
    cells=(24, 10, 3),
)

def beam_radius(u, v, w, d):
    return 0.15 + 0.10 * w / 3.0

root = mm.octet(
    cell_map,
    beam_radius=mm.cell_field(cell_map, beam_radius),
)

viz.Render(root)
Inner and outer CAD boundaries
Selected face pair
Octet lattice between faces

Automatic alignment searches compatible UV orientation and periodic seam choices. Explicit origins and flips are available when deterministic seam placement is required. See 13_cadquery_paired_face_octet.py.

Import STEP and select a face#

pv.CADModel parses a STEP model once and retains stable face handles. Select faces by index or geometric filters such as surface type, representative normal, area range, and bounding-box intersection.

from pathlib import Path

import pyvcad as pv
import pyvcad_metamaterials as mm
import pyvcad_rendering as viz

model_path = (
    Path(__file__).resolve().parents[1]
    / "data"
    / "3d_models"
    / "conformal_loft.step"
)
model = pv.CADModel.from_step(str(model_path))

curved_faces = model.select_faces(surface_type="bspline")
surface = max(curved_faces, key=lambda face: face.area)

cell_map = mm.cell_map_from_cad_face(
    surface,
    cells=(9, 7, 3),
    height=9.0,
)
root = mm.cubic(cell_map, beam_radius=0.42, node_radius=0.48)

viz.Render(root)
Imported STEP surface
Selected B-spline face
Mapped cubic lattice

The repository example resolves the STEP path relative to its own file and is available as 14_imported_step_face_selection.py.

Passing several CADFace objects to cell_map_from_cad_face(...) returns a CellMapCollection. Each face contributes an independent chart, and the mapped builder combines their results into one node.

Author other surface families directly#

The bridge is not limited to extrusions. Any CadQuery operation that produces a selectable face can define a map.

Lofted conical surface#

import cadquery as cq
import pyvcad as pv
import pyvcad_metamaterials as mm
import pyvcad_rendering as viz

cad = (
    cq.Workplane("XY")
    .circle(18.0)
    .workplane(offset=32.0)
    .circle(10.0)
    .loft(combine=True, ruled=True)
)
surface = pv.CADModel.from_cadquery(cad.faces("%CONE")).faces[0]
cell_map = mm.cell_map_from_cad_face(
    surface,
    cells=(24, 10, 1),
    height=3.0,
)
root = mm.bcc(cell_map, beam_radius=0.22, node_radius=0.28)

viz.Render(root)

Spherical band#

import cadquery as cq
import pyvcad as pv
import pyvcad_metamaterials as mm
import pyvcad_rendering as viz

cad = cq.Workplane(
    obj=cq.Solid.makeSphere(
        22.0,
        angleDegrees1=-60.0,
        angleDegrees2=60.0,
    )
)
surface = pv.CADModel.from_cadquery(cad.faces("%SPHERE")).faces[0]
cell_map = mm.cell_map_from_cad_face(
    surface,
    cells=(14, 7, 1),
    height=4.5,
)
root = mm.schwarz_p(cell_map, wall_thickness=0.45)

viz.Render(root)

Parametric freeform surface#

import math

import cadquery as cq
import pyvcad as pv
import pyvcad_metamaterials as mm
import pyvcad_rendering as viz

def saddle(u, v):
    x = 42.0 * (u - 0.5)
    y = 34.0 * (v - 0.5)
    z = 7.0 * math.sin(math.pi * (u - 0.5)) * math.sin(math.pi * (v - 0.5))
    return x, y, z

cad = cq.Workplane("XY").parametricSurface(
    saddle,
    N=20,
    tol=0.02,
    smoothing=None,
)
surface = pv.CADModel.from_cadquery(cad).faces[0]
cell_map = mm.cell_map_from_cad_face(
    surface,
    cells=(12, 10, 1),
    height=2.5,
)
root = mm.diamond(cell_map, beam_radius=0.24, node_radius=0.3)

viz.Render(root)
CadQuery-authored saddle
Parametric saddle face
Conformal diamond lattice

17_cadquery_freeform_diamond.py uses parametricSurface(...), demonstrating that the same workflow applies to a surface authored from a Python function.

Validation is always final#

Constructing a map runs CellMap’s Jacobian validation, which is the authoritative check on the extruded or interpolated volume:

  • A height larger than a concave face’s curvature radius folds the offset — “Surface CellMap is folded or singular.”

  • Pairing faces whose parameterizations cannot be brought into correspondence raises “No valid paired-surface correspondence was found,” with per-candidate Jacobian diagnostics.

Once constructed, these mapped lattices are ordinary OpenVCAD nodes. They can carry the graded fields from the previous guide, combine with the mesh-derived maps of Conformal mapping triangle meshes, and can be sent to any compatible compiler.
