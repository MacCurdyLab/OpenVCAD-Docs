Functionally graded metamaterials#

Metamaterial grading can change the geometry of a repeated cell, the spacing of the cells themselves, or attributes such as modulus and material distribution. The map-first API makes the coordinate system of each field explicit.

This guide assumes the field and attribute concepts from Functional Grading. See Python functions for attributes when a callback needs the native Numba path.

Choose world or cell coordinates#

mm.world_field(...) receives (x, y, z, d) in world coordinates. mm.cell_field(...) receives (u, v, w, d) in logical cell coordinates. In both cases, d remains the world-space signed distance to the mapped geometry.

radius_world = mm.world_field(
    lambda x, y, z, d: 0.3 + (x + 22.0) / 44.0
)

radius_cell = mm.cell_field(
    cell_map,
    lambda u, v, w, d: 0.3 + 0.2 * w,
)

World coordinates are useful when a design requirement is tied to the surrounding part or build direction. Cell coordinates are useful when a rule should travel with a rectangular or conformally deformed map.

Grade TPMS thickness and level#

def wall_thickness(x, y, z, d):
    return 0.6 + 2.0 * (x + 24.0) / 48.0

root = mm.gyroid(
    cell_map,
    wall_thickness=mm.world_field(wall_thickness),
)
The wall grows continuously from 0.6 mm to 2.6 mm in world X.

Solid TPMS structures accept the same kind of field through level. A level field changes the local balance between the two labyrinth regions rather than adding wall thickness.

The runnable thickness example is 05_graded_thickness_tpms.py.

Grade cell spacing#

Cell size belongs to the map rather than the unit cell. warp_cell_map(...) creates a new immutable map whose relative U, V, and W spacing is controlled by cell-space fields.

def u_spacing(u, v, w, d):
    return -0.8 + 1.6 * u / 8.0

warped_map = mm.warp_cell_map(
    base_map,
    u_scale=mm.cell_field(base_map, u_spacing),
    lock_u_bounds=True,
)
root = mm.gyroid(warped_map, wall_thickness=1.4)
Relative spacing weights redistribute the same eight U cells while the outer bounds remain fixed.

The scale fields are relative expansion and compression weights, not direct millimetre cell sizes. Locked axes preserve their opposing boundaries while the interior cell spacing changes. See 06_graded_cell_size.py.

Grade beams and joints#

beam_radius and node_radius accept constants or mapped fields. They remain world-unit radii even when evaluated through a curved map.

root = mm.bcc(
    cell_map,
    beam_radius=mm.world_field(beam_radius),
)
Beam radius increases continuously across the mapped BCC lattice.

See 07_graded_strut_radius.py.

Evaluate attributes in cell coordinates#

mm.cell_attribute(cell_map, attribute) adapts an existing float, vector, color, or volume-fraction attribute so its normal x, y, and z inputs receive logical U, V, and W instead.

modulus_in_cells = pv.FloatAttribute("1e6 + 2e6*x/4")
root.set_attribute(
    pv.DefaultAttributes.MODULUS,
    mm.cell_attribute(cell_map, modulus_in_cells),
)

This is useful when material intent must remain attached to cell phase after the map is warped or conformed.

Grade material distribution#

Material fields use the same OpenVCAD VOLUME_FRACTIONS attribute as any other design.

Skin and core through a TPMS wall#

The signed-distance input makes it possible to assign a tough skin and compliant core through each TPMS wall.

A clipped attribute view reveals the core material under the outer wall skin.

08_skin_core_tpms_material.py defines the two volume-fraction callbacks directly. The mm.skin_core_tpms(...) helper packages the same pattern for constant wall thickness.

Material tags on graph edges#

GraphUnitCell can carry a material tag for every edge. Those tags repeat with the unit cell and remain inspectable without expanding the lattice into separate strut nodes.

Horizontal and vertical edge groups use different material IDs across every repeated cell.

See 09_per_strut_material_lattice.py. For rule-based tagging from cell geometry, mm.per_strut_material(...) builds a tagged cell before constructing the mapped lattice.

The same geometric and attribute fields work on surface-derived maps. Continue with Conformal mapping CAD surfaces or Conformal mapping triangle meshes.
